Issue Report: Disk I/O Latency Spikes on SSDs

Issue Summary

HSD No.: 4768392001

Title: Unexpected Disk I/O Latency Spikes on SSDs Under Load

Description: The system experiences significant disk I/O latency spikes when using SSDs under moderate load. These latency increases cause delays in reading/writing data, affecting overall system performance.

Hardware Configuration

  • Hardware Model: HP ProLiant DL380 Gen10
  • Storage Device: Samsung PM883 960GB SSD
  • Memory: 128 GB RAM
  • CPU: Intel Xeon Silver 4214 (2x)
  • Network Adapter: Mellanox ConnectX-4 25G Ethernet
  • Firmware Version: 1.7.3

Software Configuration

  • Operating System: CentOS 8.4
  • Kernel Version: 4.18.0-305.el8.x86_64
  • File System: XFS
  • Disk I/O Scheduler: bfq
  • Monitoring Tool: iostat 11.5
  • Virtualization: None

Steps to Reproduce

  1. Install CentOS 8.4 on the HP ProLiant DL380 server.
  2. Configure the Samsung PM883 SSD with the XFS file system.
  3. Set the I/O scheduler to bfq.
  4. Begin moderate read and write operations using dd or other disk I/O tools.
  5. Monitor the latency with iostat.

Expected Result

The disk I/O operations should maintain low latency, even under moderate load, without any significant spikes.

Actual Result

Latency spikes of up to 500ms occur during disk write operations, significantly affecting performance.
